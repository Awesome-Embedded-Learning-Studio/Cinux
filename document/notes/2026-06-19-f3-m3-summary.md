## 产出（闭环）

- **进程组/会话身份**:Task 加 `pgid`/`sid`/`session_leader`/`controlling_tty`;`inherit_process_identity`(root fork `parent->pgid==0` 自成组,否则继承父)集中 fork/clone 继承规则。
- **身份操作 + 信号广播**:`process_group.{hpp,cpp}` 的 `setpgid`/`getpgid`/`getsid`/`setsid`(纯字段,setsid EPERM 若已是 leader);`signal.cpp` 的 `killpg`(遍历 pid registry 按 pgid 广播,pgid=0 解析调用者组);闭环 `sys_kill` pid<0(原 `TODO(F3-M3)`)。
- **4 syscall + libc wrapper**:`setpgid=109`/`setsid=112`/`getpgid=121`/`getsid=124`,注册 syscall 表 + user/libc wrapper。
- **waitpid 阻塞 + reap 契约**:批4a `sys_exit` Dead→Zombie + dequeue(对齐 exit_current)+ schedule 跳 Zombie(`pick_next` 不查 state,Zombie 留 queue 会被设 Running 崩);批4b waitpid 默认 block / `kWaitNoHang` 非阻塞 + 重扫 loop + exit 唤醒 `waiting_for_child` parent + terminal/test 全改 WNOHANG。

## 关键 GOTCHA

- **#21 `Scheduler::current()` 读 static `current_`,非 `g_per_cpu.current`**:单测设 current 必须用 `Scheduler::set_current(&t)`(两者都设),勿直接 `g_per_cpu.current=&t`。test_clone 的 futex 侥幸(futex 不经 current())掩盖此层。
- **R1 waitpid 默认阻塞挂死**:改默认阻塞前必 `grep` 全调用点确保 WNOHANG/zombie 就绪。`test_waitpid_not_exited`(child Running)实锤会挂死。
- **隐藏依赖链 → 拆批**:waitpid 阻塞的正确性依赖 exit Zombie(原 sys_exit 设 Dead,生产 reap 不到真 child)+ scheduler 跳 Zombie + terminal WNOHANG,远超 propose 的"block+唤醒"。执行中发现,拆批4a(契约)/4b(阻塞)降风险,契约先行 —— propose-then-execute 的合理回退点。
