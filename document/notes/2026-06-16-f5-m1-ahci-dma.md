# 2026-06-16 F5-M1 AHCI DMA 迁移 — 里程碑总结

> 闭环 block device→AHCI DMA 栈：M3 DmaPool/PrdtBuilder 真正落地到 AHCI 驱动。

## 目标

收编 ahci.cpp 内部 ad-hoc DMA（command list/FIS/command table 的手动 `g_pmm`+`g_vmm.map`+硬编码 `+0xFFFFFFFF80000000ULL` + execute_command 手动 PRDT）→ M3 `DmaPool`/`PrdtBuilder`；补 `AHCIBlockDevice` 的 `block_count()`（ATA IDENTIFY）+ `flush()`（FLUSH CACHE）M4 占位。

## 产出（改 `kernel/drivers/ahci/`）

| 改动 | 文件 | 内容 |
|------|------|------|
| setup_port DMA 收编 | ahci.cpp/hpp | command list+tables/FIS 手动 PMM+VMM+硬编码偏移 → `g_dma_pool` DmaBuffer per-port（`cmd_list_buf_`/`fis_buf_`，替 `cmd_list_phys_`/`fis_buf_phys_`）；删未用的 MMIO_VIRT_BASE 映射 |
| PRDT scatter-gather | ahci.cpp | `execute_command` 手动 `prdt[0]` → `PrdtBuilder`（拆 4MB 段，循环填 prdt，IRQ 置最后段） |
| IDENTIFY/FLUSH | ahci.cpp/hpp + ahci_config | `execute_command`/`build_cfis` 参数化 `command` byte；`AHCI::identify`（0xEC，words 60-61 28-bit 容量）+ `AHCI::flush`（0xEA 无 data） |
| AHCIBlockDevice 接 | ahci_block_device.hpp/cpp | `create` 时 identify 填 `capacity_blocks_`（M4 占位 0→真值）；`flush()` override 调 `AHCI::flush`（替默认空） |

## 关键设计

- **布局保持**（批1）：command list+tables 同 4KB DmaBuffer（32 headers + command table），FIS 单独。只换 DMA 来源（direct-map virt=phys+KERNEL_VMA 等价旧硬编码偏移），不动 AHCI 硬件要求的结构布局。低风险。
- **execute_command 参数化**（批3）：`bool write_cmd` → `uint8_t command`，read/write/identify/flush 统一经一条命令路径（identify=0xEC 读 1 sector，flush=0xEA 无 data，PRDT `add(0,0)`→0 段）。
- **GOTCHA #7**（direct-map 勿 unmap）：command list/FIS DmaBuffer RAII release 只 free phys（direct-map PTE 永久，同 M3 DmaPool）。

## 验证

run-kernel-test **705/0**（F5-M1 重构+新功能，测试数不变）+ test_host 绿 + **FORTIFY 对等**（本地 `make -C build run-kernel-test` + `test_host` 复现 CI，CMakeLists RELEASE 显式 `_FORTIFY_SOURCE=2`）。关键：`AHCIBlockDevice::block_count() > 0` 真机过 = QEMU AHCI ATA IDENTIFY 工作正确（28-bit 容量 words 60-61 解析对），flush 真命令不崩。

## 遗留

- **中断驱动**（todo 目标 3）：仍轮询 CI 位，留后续（非 F5-M1 范围，propose 时已排除）。
- **48-bit 容量**：当前 28-bit（words 60-61），ext2.img 小够用；大磁盘升 words 100-103。
- **F5-M2 VirtIO / M3 NVMe**：下一设备驱动（复用 `IBlockDevice` + DmaPool/PrdtBuilder 契约）。
