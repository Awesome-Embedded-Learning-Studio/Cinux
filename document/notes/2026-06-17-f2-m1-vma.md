# F2-M1: VMA 区域记账

> 日期 2026-06-17 · 里程碑 F2-M1 · 分支 `feat/f2-m1-vma`
> 为 `AddressSpace` 引入 VMA(Virtual Memory Area)记账，为 mmap/brk/demand paging/CoW 提供单一事实源。

## 背景

F2 内存管理增强的根节点。F1 内核基建完成后，F2 是 ROADMAP 点名的最大结构瓶颈(阻塞 mmap/brk/Page Cache/COW/共享内存)。VMA 是整条 F2 链的地基。M1 范围 = 记账 + 让 PF handler 查询(诊断)；mmap(M2)/brk(M3)/Page Cache(M4)/demand paging 重构(M5)留后续。

## 设计

- **数据结构**：`LinkedListVMAStore`(侵入式双向链表，按 start 有序，insert 合并相邻同权限 / remove 拆分中间)。`IVMAStore` 抽象接口(可换红黑树)。store-owns 节点(RAII)。
- **错误模型**：`ErrorOr`(A.6)报逻辑错误(InvalidArgument 坏范围 / AlreadyExists 重叠)。OOM 不建模为可恢复——operator new 返 nullptr(kernel 禁异常)，沿用 `new AddressSpace()` 同款崩惯例(heap 自动 expand)。
- **AddressSpace 集成**：直接对象成员 `LinkedListVMAStore vma_store_` + `Spinlock vma_lock_`(非指针，RAII)。补 `LinkedListVMAStore` move(AddressSpace 是 move-only，值成员须可 move)。
- **注册点**：execve PT_LOAD 段(PF_W/PF_X→VmaFlags) + init/gui_init 用户栈(Stack flag)。insert 失败→路径失败保 VMA 完整。
- **PF 查询**：demand paging 加 `find()` 交叉检查(诊断增强，未命中 warn 但仍 demand page，不改行为)。

## 关键决策

1. **PF 校验选"诊断增强"非"硬门控"**：侦察发现现有 demand paging 对任何 not-present 宽松映射(野指针/栈下方都给页)。"无 VMA→真 segfault"需重构 demand paging(VMA 门控 + Stack 自动下扩 + guard)，漏判→shell halt，高风险。M1 选诊断(零风险)，真 segfault 留 M5 demand paging 重构。
2. **用户布局常量按实际栈顶校正**：todo 草案 `USER_MMAP_END=0x7F0000000000`(127TB)越过实际 `USER_STACK_TOP≈32GB`，不自洽。改为 mmap[4GB,24GB)，栈顶下留 8GB gap。
3. **backing 字段用真实类型 `InodeOps*`**：todo 草案写 `fs::Inode*` 但 inode.hpp 实际是 `class InodeOps`(无 `class Inode`)。forward-decl InodeOps。

## 陷阱

- **operator new 返 nullptr 非 panic**：crt_stub 的 operator new 直接 `return g_heap.alloc()`，失败返 nullptr(不是 kpanic)。普通 new 在 null 上构造→崩。VMA 节点小 + heap expand，OOM 实际不发生，沿用惯例。
- **klog_warn 是宏非命名空间函数**：不能写 `cinux::lib::klog_warn`(展开成 `cinux::lib::::cinux::...` 语法错)。用裸 `klog_warn(...)`。
- **位运算符须 constexpr**：test 用 `constexpr VmaFlags kRw = Read|Write`，`operator|` 须加 constexpr。
- **启动路径不被 run-kernel-test 覆盖**：run-kernel-test 跑 test harness kernel_main，不经 init.cpp shell 启动(execve+栈注册 VMA)。批3 的 VMA 注册执行靠实机冒烟 + 单测逻辑。
- **grep -v "^\[" 误过滤内核日志**：`[INIT]`/`[EXECVE]`/`[VMM]` 等都以 `[` 开头，实机冒烟 grep 要保留。

## 验证

- run-kernel-test：705→712(+7：批1 VMA store +5，批2 AddressSpace 集成 +2)，全绿。
- 全量 build(含 host)：CI 对等(批2 改 address_space.hpp 公共头)。
- clang-format：本地 v22(CI format job 已禁用)。
- 实机冒烟(timeout make run)：启动到 GUI worker 无 panic/halt/#PF(不炸)。GUI shell 需鼠标交互未触发，execve VMA 注册靠 run-kernel-test + LinkedListVMAStore 单测覆盖。

## 遗留

- PF 真 segfault(硬门控)：留 M5 demand paging 重构。
- fork 复制 VMA store：fork 子进程当前 new 空 AddressSpace(VMA 不复制)——F3 进程或后续。
