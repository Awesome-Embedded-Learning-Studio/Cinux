# F2-M6: ext2 Cache（read 经 PageCache）

> 日期 2026-06-17 · 里程碑 F2-M6 · 分支 `feat/f2-m6-ext2-cache`
> `sys_read` 对磁盘文件走 PageCache——读路径与 demand paging 共用同一份缓存，重复读命中免读盘。

## 背景

M4 的 PageCache 只服务 **file-backed mmap 的 demand paging**（PF handler 经 `get_page` 填充），而 `sys_read` 对普通文件直走 `Ext2FileOps::read`——**每次按 ext2 块读盘，无缓存**。于是同一文件"mmap 读"与"read() 读"各走各的，read() 重复读同一页反复 I/O。M6 让 `read()` 也经 PageCache，闭环"读路径唯一缓存层"：`read()` 与 demand paging 共用 `(Inode*, page_offset)` 缓存。

## 设计

- **`PageCache::read_bytes(inode, file_off, buf, count)`**（[page_cache.hpp](../../kernel/mm/page_cache.hpp) / [page_cache.cpp](../../kernel/mm/page_cache.cpp)）：按页对齐切片，每页调既有 `get_page(inode, page_base)`——命中 bump ref、未命中 alloc 页 → 锁外 `inode->ops->read` 填充 + EOF 零填 → 锁内 insert（M4 既有逻辑）——再 `memcpy` 切片到用户 buf，`release`。EOF 用 `inode->size` 截断（ext2 lookup 已填 `vfs_inode.size = disk.i_size`）。read() 缓存路径原语。
- **避免递归**：`read_bytes → get_page → Ext2FileOps::read`（读盘原语，不改）；`Ext2FileOps::read` 不调 page_cache。链终止——不存在"read 调 cache、cache 调 read"的环。
- **判别磁盘文件 vs pipe**（`sys_read` 分流）：pipe 的 inode `type` 也是 `InodeType::Regular`（[test_sys_pipe.cpp](../../kernel/test/test_sys_pipe.cpp):105），type 不可靠；禁 RTTI（DIRECTIVES A）故不能 `dynamic_cast`。在 `InodeOps` 加 `virtual bool is_page_cacheable() const`（默认 false，[inode.hpp](../../kernel/fs/inode.hpp)），`Ext2FileOps` override 返 `true`（[ext2_common.hpp](../../kernel/fs/ext2_common.hpp)）。pipe/ramdisk/测试 mock 继承默认 false，行为不变。
- **sys_read 分流**（[sys_read.cpp](../../kernel/syscall/sys_read.cpp)）：FDTable 命中后，`inode->ops->is_page_cacheable()` 为真 → `g_page_cache.read_bytes(...)`，否则原 `inode->ops->read(...)`。offset 推进 + errno 翻译不变。

## 关键决策

1. **判别用 virtual `is_page_cacheable()`（非 type 字段）**：pipe 与 ext2 文件 type 都是 Regular，禁 RTTI 无法 dynamic_cast。virtual 带默认实现是源码兼容的——现有子类（PipeReadOps/PipeWriteOps/RamdiskFileOps/测试 mock）继承默认 false 无需改；只 Ext2FileOps override true。语义贴 ops vtable 既有的"每 fs 能力"模式。
2. **read_bytes 切片复用 get_page**：不另起一套缓存逻辑，直接复用 M4 的 `get_page`（填充/命中/refcount/EOF 零填全现成）。read_bytes 只负责"把跨页的 [off,off+count) 按页切片，从缓存页 direct-map virt memcpy"。
3. **EOF 以 `inode->size` 为准**：read() 语义要求返回到 EOF 为止的真实字节数。get_page 内部已对页尾零填，但 read_bytes 必须在 `inode->size` 处停（否则越 EOF memcpy 零字节仍返回 count，错）。ext2 lookup 填 size 已核实。

## 陷阱

- **pipe/ramdisk 的 type 也是 Regular**：用 `inode->type` 判是否走缓存会误把 pipe/ramdisk 读路由进 page_cache（pipe 内容非磁盘，缓存即错）。必须用 `is_page_cacheable()` virtual 判别。批1 单测用 FakeFileOps（默认 false）直调 `cache.read_bytes` 绕过此判别，故批1 不覆盖判别；批2 端到端 ext2 测试才覆盖。
- **read_bytes 的 partial-read 契约**：中途某页 `get_page` 失败（OOM/读错），已读字节 > 0 时返回已读量（不回传错），仅 0 字节时回传 error——匹配 POSIX read() 的部分读语义。
- **递归规避**：read_bytes 是新函数，Ext2FileOps::read（get_page 的填充原语）不改、不调 page_cache。若误把缓存逻辑塞进 Ext2FileOps::read 会形成 read→get_page→read 死循环。
- **`g_page_cache.hit_count()` 跨测试累积**：它是 boot 全局，整个 suite 只 init 一次。端到端测试断言"第二读 hit_count 上升"时，须在第二读**前**捕获基线 `hits_mid`（而非 suite 起始），避免先前测试读过同文件污染断言。

## 验证

- run-kernel-test：**730 → 734**（批1 +3：read_bytes 基本正确+EOF / 二读命中缓存（读盘次数不增+hit_count 增）/ 跨页 EOF 裁剪；批2 +1：真 AHCI/ext2 读 /hello.txt 两遍 hit_count 上升 + 字节一致 = sys_read→read_bytes 接线端到端铁证）。
- host `test_host`：**49/0**（InodeOps 加 virtual 是源码兼容，ext2_inode_ops/syscall_ext2/sys_pipe/shell_redirect 等 mock 子类继承默认 false，编译+运行过——L5 CI 对等盲区覆盖）。
- 实机冒烟（`cmake --build build --target run`，生产内核 serial）：启动 ext2 挂载→VFS /→Multi-Terminal→GUI Desktop→Mouse→Desktop icons(Shell,Calculator)→PIT tick 到桌面，**无 panic/halt/segfault**（仅早期 int $3 断点自测）。shell 经 sys_read→read_bytes 读 ext2 文件不炸。

## 遗留

- **脏页写回 / MAP_SHARED 写一致性**：M6 只做读路径缓存（沿用 M4 MVP 范围）。read() 读到的缓存页与磁盘写不同步的问题，留后续（write() 经缓存 + 回写）。
- **缓存淘汰（LRU）**：PageCache 无淘汰（ref_count 不归零页常驻），长时间运行可能膨胀。留后续。
- **跨进程共享缓存 + CoW-for-shared-file**：当前每进程 PF 独立映射缓存页物理地址（共享同 phys），但无 CoW 写时复制隔离。留 F3。
- **kprintf 全量迁移 / read 路径埋点**：read_bytes 未加调试埋点（保持洁净）；若需观测命中率可加 stats 暴露。
