# F9 批4:启用 SMAP(CR4 bit 21)+ stac/clac 全入口接入

> 2026-06-25 · F9 安全机制 · 批4 · `feat/f9-security` · commit(本次)
> M1 收官:SMEP(批3)+ SMAP(批4)补齐内核↔用户页双向硬件隔离。

## 背景

SMAP(Supervisor Mode Access Prevention,CR4 bit 21)阻止内核读写用户页——内核访问用户内存必须显式 STAC(置 AC flag)临时放行,CLAC 关闭。配合 SMEP(管执行)形成内核/用户页完全隔离。这是 M1 硬骨头:开 SMAP 前必须给所有访用户入口加 stac/clac,否则内核一碰用户内存就 #PF。

## 改动

- **[paging.cpp](../../kernel/arch/x86_64/paging.cpp) / [paging.hpp](../../kernel/arch/x86_64/paging.hpp)**:批3 的 `enable_smep()` 扩展为 `enable_smep_smap()`——一次 CPUID.07H:EBX 查 SMEP(bit 7)+ SMAP(bit 20),分别设 CR4[20]/[21](无 CPUID 支持则跳过,写不支持位会 #GP)。
- **[syscall.S](../../kernel/arch/x86_64/syscall.S)**:SYSCALL entry swapgs 后 `stac`(handler 可访用户),exit `clac` + sysretq。SYSCALL 必从 user 来,无条件 stac/clac。
- **[interrupts.S](../../kernel/arch/x86_64/interrupts.S)**:三个 ISR 宏(NOERRCODE/ERRCODE/IRQ)的**条件 swapgs 块**加 stac(entry)/clac(exit)——只从 user 进来才置 AC(kernel-mode 中断继承 AC,不破坏 copy_from_user 期嵌套)。
- **[main.cpp](../../kernel/main.cpp) / [ap_main.cpp](../../kernel/arch/x86_64/ap_main.cpp)**:`enable_smep_smap()`(BSP + 每 AP,CR4 per-CPU)。

## 关键决策

- **AC flag 条件置位(同 swapgs 块)**:stac/clac 挂在 `testb $3,%al` 的 user-mode 分支,与 swapgs 同条件。kernel-mode 中断(如 PIT 打断 copy_from_user)不动 AC,返回后 kernel 的 AC 状态延续——无条件 stac/clac 会清掉 kernel 的 AC,破坏嵌套。
- **stac/clac 在 SMAP 关时是 NOP**:先加指令(SMAP 关 NOP,行为不变,931/0)再开 SMAP,两步隔离风险(asm 对不对 / SMAP 覆盖全不全分开验证)。
- **ISR 宏三处同步**:NOERRCODE/ERRCODE/IRQ 共享 swapgs 纪律;int $0x80 sigreturn(NOERRCODE)读用户栈 SignalFrame 走 entry stac 护。

## DEBT-019 重审

SMAP 开后,validate_user_ptr + 直接解引用(DEBT-019 的 PF 兜底)现在**双层保护**:syscall entry stac 显式放行访用户(合法路径),漏 stac 的意外访用户 → SMAP #PF(而非 PF 兜底默默通过)。validate 仍只查 canonical 地址(行为不变);改 copy_from_user 的完整修复留 F10。SMAP 让 DEBT-019 的边界更显式、更安全。

## 验证

- run-kernel-test **931/0**(SMAP 开,无 SMAP #PF)。
- `cmake --build build --target run` 冒烟:GUI Desktop / shell / xHCI keyboard 全正常,零 panic/#PF。syscall entry stac 护住 shell 读用户内存这条关键路径。
