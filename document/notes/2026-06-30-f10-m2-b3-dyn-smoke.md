# F10-M2 批3 — 端到端动态 hello ring-3 smoke + 收官

> 2026-06-30。批1(kernel PT_INTERP 路径)+ 批2(动态工具链 + ext2 装 interp)
> 之后,本批在 QEMU 真内核里 fork+execve("/hello-dyn") 端到端验证动态链接,
> 并收官 F10-M2。分支 `worktree-f10-m2-dynlink`。

## 背景 / 目标

把动态 ELF 跑起来:smoke worker fork → child 装 AS → launch_user_program("/hello-dyn")
→ execve 识别 PT_INTERP → 加载 interp(ldso)→ musl ldso 重定位主程序 → 跳 AT_ENTRY →
write() 经 libc.so 输出。父 waitpid,以 exit 0 为通过。单核 + -smp 2 两 leg。

## 设计 / 决策

- **smoke harness 复用**:静态 /hello(F10-M1)与动态 /hello-dyn 共用同一个
  `musl_hello_smoke_entry`(scheduler/ext2 脚手架一份)。外层 gate 改
  `#if defined(CINUX_MUSL_HELLO_SMOKE) || defined(CINUX_MUSL_DYN_SMOKE)`,内层两相各自
  `#ifdef CINUX_MUSL_HELLO_SMOKE` / `#ifdef CINUX_MUSL_DYN_SMOKE` 独立 gate,互不耦合
  (可单开 DYN_SMOKE 跑动态相)。`ap_test_selfcheck` + 注册段的 gate 同步改成任一开则 AP 参与。
- **`CINUX_MUSL_DYN_SMOKE` option**(默认 OFF):顶层 CMakeLists.txt + kernel/CMakeLists.txt。
  CI 无 sysroot 不受影响;本地 `cmake -DCINUX_MUSL_DYN_SMOKE=ON`。
- dyn 相 5 迭代(fork+execve+waitpid,每次都重载 interp + 重定位,比静态重,故少于静态的 20)。

## 关键坑(本次最大)

- **ext2 不支持 double-indirect 块** → interp(822KB)读不到。现象:
  `[ELF] segment read failed at offset 274432`。274432 = block 268 = ext2 1024-byte 块下
  direct(12)+single-indirect(256)的**双重间接起点**。`ext2_common.cpp` 块映射只处理
  direct + single-indirect,double-indirect 直接 `break`(截断)。静态 /hello 小(几 KB)从没触发。
  **解法**(最小、不动 kernel):create_ext2_disk.sh 改 **4096-byte 块**(single-indirect 上限
  ≈4MB,822KB interp 落进单间接)。`block_buf_[4096]` 本就支持 max ext2 block。image 4MB→8MB。
  ext2 驱动动态读 block_size,自适应。**double-indirect 缺失登记为 ext2 follow-up**(见下)。
- **Read 工具 vs Bash sed/git 缓存不一致**:中途 Read 显示编辑在、sed/git 显示原态,实为 Bash
  工具缓存陈旧输出;以 **python 直读磁盘** 为准(编辑其实已落盘)。下次遇类似不一致,python 裁决。

## 验证

- `cmake -DCINUX_MUSL_DYN_SMOKE=ON -DCINUX_MUSL_HELLO_SMOKE=OFF -DCINUX_BUILD_TESTS=ON`,
  `timeout 50` 单 leg(wrapper + `-vnc :1`,避并行 worktree 占 5900):
  - 串口:`[EXECVE] loaded interp /lib/ld-musl-x86_64.so.1 base=0x10000000 entry=0x10072530`
    + **5× `Hello from musl on CinuxOS!`** + `[F10-M2] smoke: hello-dyn 5/5 iters PASS -> PASS`。
  - **968 passed, 0 failed** + ALL TESTS PASSED,wrapper exit 0。
- **-smp 2 leg**:968/0 + AP1 online + AP readback PASS + 5× Hello + hello-dyn 5/5 PASS,exit 0。
- 默认 OFF(batch 1 已验 968/0)零回归。

**结论**:**CinuxOS 能跑 musl 动态用户程序**。kernel 加载 PT_INTERP 指定的 interp、
喂对 auxv,符号解析/GOT/PLT/DT_NEEDED 全由 musl ldso 在用户态完成。

## 范围栅栏 / follow-up

- **ext2 double-indirect / triple-indirect 缺失**(本次发现):>268KB(1024 块)/ >4MB(4096 块)
  的文件 ext2 读会截断。4096 块是合法配置,但真修在 ext2 块映射补双/三重间接(留 F2-M6/F6)。
- ELF base ASLR / PIE 主程序(批1 的 ET_DYN 接受已铺路,重定位 R_X86_64_RELATIVE 留后)。
- interp base ASLR(现固定 USER_INTERP_BASE,配 PIE 时一起做)。
- glibc 动态二进制(PT_INTERP=/lib64/ld-linux-x86-64.so.2 天然支持,按需验)。
