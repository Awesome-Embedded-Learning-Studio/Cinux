# F9 批8 — ASLR 内核侧三件套(栈/mmap/brk 地址随机化)

> 2026-06-26 / F9 安全机制 / 批8(M2 ASLR)。分支 `feat/f9-security`,commit `26e8bfa`。
> 范围(用户拍板):**内核侧三件套**——栈顶 + mmap 起点 + brk 起点随机化。ELF base 随机化拆后续。

## 背景:为什么 ELF base 不能直接随机化

批8 启动前的调研铁证(决定打法):

用户程序 shell 是 **non-PIE**:
- `readelf -h build/user/shell`:`Type: EXEC`(ET_EXEC=2),`Entry: 0x400270`,单 PT_LOAD `p_vaddr=0x400000`。
- 编译选项:`-fno-pie -no-pie -mcmodel=small -static -nostdlib`(`user/CMakeLists.txt`),`user/linker.ld` 固定 `USER_VMA=0x400000`。
- 反汇编:`cmd_help` 里 `mov $0x401488,%edi`(字符串字面量地址,32-bit 绝对立即数)。全程序 **0 处 RIP-relative、99 处 `0x40????` 绝对引用**。
- `readelf -r`:no relocations(静态无重定位)。

结论:non-PIE small-model 把全局/字符串地址烤成 32-bit 绝对立即数。ELF base 偏离 0x400000 → 这些绝对引用全错位 → 必崩。「直接给 p_vaddr/e_entry 加 offset」的捷径是**死路**。

ELF base 随机化的两条真路(都非捷径,留后续单独 propose):
- 路 A:shell 改 PIE(ET_DYN)+ 内核 ELF loader 加动态重定位处理(apply `R_X86_64_RELATIVE`,当前 execve 只 copy 段、不处理 `.rela.dyn`)+ 放开 `kernel/proc/elf_types.cpp:45` 的 `e_type != ET_EXEC` 拒绝。
- 路 B:不做 text 随机化(本批选择)。

## 本批改动(内核侧三件套,纯内核零碰 ELF loader)

新增 `kernel/lib/aslr.hpp`:3 个 inline 偏移函数,单一权威,喂批7 `g_random`(xoshiro256**):
- `aslr_stack_offset()`:`& 0x7FF000`(0..8 MiB,页对齐)
- `aslr_mmap_offset()`:`& 0x3FFFF000`(0..1 GiB,页对齐)
- `aslr_brk_offset()`:`& 0xFFF000`(0..16 MiB,页对齐)

三处消费:
- **栈顶**(`kernel/proc/user_launch.cpp::launch_user_program`):局部 `stack_top = USER_STACK_TOP - aslr_stack_offset()`,替 4 处引用(pre-map base / Stack VMA start-end / jump RSP)。偏移页对齐 → x86_64 ABI `USER_ABI_RSP_OFFSET` 的 `%16==8` 不变量保住(`usermode.hpp` 的 static_assert 仍成立,因减页对齐值不改变 %16)。
- **mmap**(`kernel/syscall/sys_mmap.cpp`):非 MAP_FIXED 分支 `find_free_area(USER_MMAP_BASE + aslr_mmap_offset(), ...)`。窗口边界 [4GB, 24GB) 不变;MAP_FIXED 仍依用户指定地址。
- **brk**(`kernel/proc/execve.cpp`):`brk_initial = align(max_seg_end) + aslr_brk_offset()`,clamp 不超 `USER_BRK_MAX`(64 MiB),保 heap 余量。

## 牵连核实(改前 grep 引用方)

- `USER_STACK_TOP` 生产引用**只在 `launch_user_program` 一个函数**(4 处全局部);PF 栈增长走 Stack VMA(F2-M5 硬门控)不查常量 → **无需动 Task 结构**,改一个函数的局部变量即可。
- `USER_MMAP_BASE/END` 生产只在 `sys_mmap`;现有 test 断言 `r >= USER_MMAP_BASE` 仍成立(hint = base + offset ≥ base)。MAP_FIXED test(`test_mmap.cpp`)在窗口内仍允许。
- `brk_initial` 字段已有(execve 设);`test_brk.cpp` 自设 `brk_initial = USER_BRK_BASE`,不走 execve,不受影响。

## 机制验证

- `kernel/test/test_aslr.cpp`(4 项,run-kernel-test):三函数各 64 次抽样断言**页对齐 + 区间**;16 次抽样断言**非全同**(每次调 `next64` 推进 xoshiro 状态,必变;恒定 = helper 坏)。
- run-kernel-test **936/0**(932 + 4 test_aslr)。
- `make run` 冒烟两次:**零 panic**(boot 到 GUI Desktop + xHCI USB keyboard armed)。

## 诚实记录(GOTCHA)

- GUI 模式 shell 不自动启动(需点 Shell 图标),故 make run 不触发 `launch_user_program`,无 `[PROC] jumping stack_top=` 输出。栈顶随机化的端到端「每次不同」**未在 make run 直观观测**。但 `aslr_stack_offset()` 随机性已由 test_aslr 验证,`launch_user_program` 必用之,逻辑确定生效。(非 GUI 构建会自动 fork shell,可观测量化,本批未跑。)
- test log 含 ANSI escape → grep 当二进制静默(F4-M1 GOTCHA 复现),查日志用 `grep -a`。
- ELF text/data base **未**随机化(non-PIE 绝对寻址铁证)。攻击面覆盖:栈溢出(栈随机)+ mmap(mmap 随机)+ heap(brk 随机);**text ROP 防护缺**,留 PIE follow-up。
