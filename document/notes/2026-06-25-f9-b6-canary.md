# F9 批6:启用 Stack Canary(M4)

> 2026-06-25 · F9 安全机制 · 批6(M4 Stack Canary)· `feat/f9-security`

## 改动

- **[kernel/CMakeLists.txt:26](../../kernel/CMakeLists.txt#L26)**:`-fno-stack-protector` → `-fstack-protector-strong -mstack-protector-guard=global`。GCC 给每个有局部数组的函数加 canary prologue/epilogue,比对 `__stack_chk_guard` 全局(非 TLS %fs:0x28 —— freestanding 内核无 TLS 设置)。
- **[boot.S](../../kernel/arch/x86_64/boot.S)**:Step 2c `rdtsc` 播种 `__stack_chk_guard`(per-boot 随机,在第一个 protected 函数/ctors 前)。
- **[crt_stub.cpp](../../kernel/arch/x86_64/crt_stub.cpp)**:加 `__stack_chk_guard` 全局(canary);`__stack_chk_fail` 从 outb+halt 升级 `kpanic`(对齐 __assert_fail,backtrace/memstats 诊断)。顺带 `__cxa_pure_virtual` old-style cast → static_cast(零警告)。

## 关键点

- **-mstack-protector-guard=global**:x86_64 GCC 默认用 TLS(%fs:0x28)canary,但 freestanding 内核无 TLS 初始化 → 改 global `__stack_chk_guard`(CinuxOS 提供 + boot 播种)。
- **canary 必须在第一个 protected 函数前播种**:boot.S rdtsc(BSS clear 后、_init_global_ctors 前),ctors/kernel_main 可能是 protected 函数。
- **__stack_chk_fail → kpanic**:stack smashing 触发统一 panic(backtrace + memstats),不是裸 halt。
