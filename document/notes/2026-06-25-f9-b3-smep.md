# F9 批3:启用 SMEP(CR4 bit 20)

> 2026-06-25 · F9 安全机制 · 批3 · `feat/f9-security` · commit(本次)

## 背景

SMEP(Supervisor Mode Execution Prevention)阻止内核态(Ring 0)执行用户态(Ring 3)页面的代码——内核若意外 jump 到用户页执行(经典提权攻击),触发 #PF 而非得逞。NX(批2)管用户态执行用户数据页;SMEP 管内核态执行用户页。两者互补。

## 改动

- **[paging.cpp](../../kernel/arch/x86_64/paging.cpp) / [paging.hpp](../../kernel/arch/x86_64/paging.hpp)**:加 `enable_smep()`——CPUID.07H:EBX[7] 检测 SMEP 支持,若支持读 CR4 | (1<<20) 写回。
- **[main.cpp](../../kernel/main.cpp)**:BSP 在 `usermode_init()` 后调 `enable_smep()`。
- **[ap_main.cpp](../../kernel/arch/x86_64/ap_main.cpp)**:每个 AP 在 `usermode_init_asm()` 后调 `enable_smep()`(CR4 是 per-CPU,AP 须对齐 BSP)。

## 关键决策

- **必须 CPUID 检测**:写不支持的 CR4 位(老 CPU 无 SMEP)会 #GP 崩。SMEP 是 2011+ Intel 特性(不像 NX 是 x86_64 baseline),所以 gate。QEMU qemu64 / 现代真机都支持,检测通过即设。
- **per-CPU,BSP+AP 都设**:CR4 每核独立。AP 在 ap_main 对齐 BSP(同「AP 须匹配 BSP CPU 配置」教训类,参考 LSTAR 修复 [[smp-shell-migration-df-open]])。
- **启用安全**:CinuxOS 内核经 sysretq/iretq 切到用户态(改 CS,不直接执行用户页代码),故开 SMEP 不崩。sigreturn 页(USER_SIGRETURN_PAGE)是**用户态**执行 int $0x80(ring 3),SMEP 只管 ring 0 执行用户页,不影响它。

## 验证

- run-kernel-test **931/0**(SMEP 开启后无 instruction-fetch #PF)。
- `cmake --build build --target run` 冒烟:GUI Desktop / shell / xHCI keyboard 全正常,**零 panic / #PF**。
