# F4-M2 批2:I/O APIC 驱动

> 2026-06-19。F4-M2 批 2。范围:IOAPIC driver(indirect IOREGSEL/IOWIN + set_redirect + mask/unmask + id/version/max)+ mock 单测。不切换(M2-3)。
> 基线 = M2-1(865/0)。

## 决策
- **indirect 访问(IOREGSEL/IOWIN)**:IOAPIC 寄存器经 IOREGSEL(0x00)选索引 + IOWIN(0x10)读/写,与 LAPIC 直接 MMIO 不同。read(reg) = base_[0]=reg; return base_[4](0x10/4)。
- **MMIO 在 LAPIC 旁**(KMEM_MMIO_BASE + 0x11000,LAPIC 在 +0x10000),FLAG_PCD 禁缓存。
- **set_redirect 64-bit 双写**:每 pin 2 寄存器(lo + hi),lo = vector | polarity | trigger,hi = dest_apic_id << 24(physical dest mode,bit11=0)。M2-3 消费 ISA IRQ override(GSI 映射)。
- **mask/unmask = read-modify-write bit16**:低字的 mask 位。
- **M2-2 mock 限制**:IOAPIC indirect 在纯 RAM mock 无法模拟"IOWIN 反映 IOREGSEL"(IOWIN 是固定位置)。故 mock 验证**写模式**(IOREGSEL 选 + IOWIN 值)+ bit16 算术;redirect low 的 vector/polarity + read-based(id/version/max)留 M2-3 真机。

## 实现
- `io_apic.hpp`:IOAPIC(bind/init/read/write/set_redirect/mask/unmask/id/version/max_redirect)+ indirect 端口/寄存器/redirect-bit 常量 + g_ioapic。
- `io_apic.cpp`:实现(init map+PCD;read/write indirect;set_redirect 双写;mask/unmask RMW bit16;max_redirect=(ver>>16)+1)。
- `test_apic.cpp`:+4 IOAPIC mock 测(indirect write 选 reg+写窗口 / set_redirect dest in high / mask bit16 / unmask 清 bit16),复用 MockMmio。

## 验证
- `timeout 40 run-kernel-test`:**869/0**(865 + 4 IOAPIC mock)。零编译警告。
