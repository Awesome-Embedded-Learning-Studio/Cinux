| 点 | 现状 | 多核问题 |
|----|------|---------|
| `current_` | `scheduler.cpp:170` 静态全局 | 每核独立 → per-CPU |

## 2. PerCpu GS 设计(Phase 1 核心)

### 当前 GS 语义(必须先理解)
- `usermode_init`(`usermode.cpp:57-69`):alloc gs 数据页,`write_msr(MSR_KERNEL_GS_BASE=0xC0000102, gs_virt)`。页内 `[0]=kernel_stack,[1]=0,[2]=scratch`。
- `syscall.S:58-59`:syscall 入口 `%gs:0`→RSP(kernel stack),`%gs:8`/`%gs:16` scratch。SYSCALL 硬件 swapgs(kernel mode GS_BASE = KERNEL_GS_BASE = gs 页)。
- `context_switch.S:61-99`:按 **per-task** 保存/恢复 GS_BASE + KERNEL_GS_BASE(CpuContext offset 64/72)。单核下所有 task 的 kgs_base = 同一 gs 页(等价)。
- `update_syscall_stack`(`per_cpu.hpp:14`):写 `gs_page_vaddr[0] = stack_top`(%gs:0)。
- FS_BASE = per-thread TLS(F3-M2),不动。

### 目标设计:GS base → PerCpu 块(per-CPU)
```cpp
struct PerCpu {
    uint64_t kernel_stack;   // @0  — syscall %gs:0(兼容!current task 的内核栈顶)
    uint64_t scratch1;       // @8  — syscall %gs:8(user RSP)
    uint64_t scratch2;       // @16 — syscall %gs:16(return value)
    Task*    current;        // @24 — 当前 task(替代 scheduler::current_)
    uint32_t cpu_id;         // @32
    uint32_t apic_id;        // @36
    // 后续:M4 per-CPU run queue 指针、统计等
};
alignas(4096) PerCpu percpu_blocks[kMaxCpus];  // 每 CPU 一块,4KB 对齐
```
- `percpu()`:kernel mode 读 `MSR_GS_BASE`(swapgs 后 = KERNEL_GS_BASE = &percpu_blocks[cpu])。
- BSP init:`wrmsr(KERNEL_GS_BASE, &percpu_blocks[0])`;AP init(ap_main):`wrmsr(KERNEL_GS_BASE, &percpu_blocks[cpu])`。
- **context_switch 改动**:不再 per-task 保存/恢复 GS(GS per-CPU 固定)。CpuContext offset 64/72(gs/kgs)可移除或保留为常量。Task.ctx.kgs_base 去掉(clone/fork/task_builder 的 `kgs_base = gs_page_vaddr` 删除)。
- **update_syscall_stack**:`percpu()->kernel_stack = stack_top`(%gs:0 自动更新,因 PerCpu@0 = kernel_stack)。

### 关键不变量(重构后必须保持)
- syscall `%gs:0` = PerCpu.kernel_stack = current task 内核栈顶。
- `current()` = `percpu()->current`(经 GS,无静态全局)。
- 单核(BSP):percpu_blocks[0] 等价旧 g_per_cpu。

## 4. 风险 + GOTCHA(执行时严守)
1. **P1-2 不可中途停**(GS 重构中间态破坏 syscall):必须一批完成 + `timeout 40 make run` 真机验证 syscall/PIT/键盘/鼠标全工作才 commit。
2. **PerCpu.kernel_stack 必须 @0**(否则 syscall `%gs:0` 读错)。static_assert offsetof(kernel_stack)==0。
3. **context_switch 去 GS 保存/恢复**后,FS_BASE(TLS)仍 per-task 保存/恢复(不动)。
4. **AP trampoline @0x8000**:必须在 <1MB 实模式可达区;QEMU SIPI vector=页号(0x08 → 0x8000)。
5. **AP 的 GDT/IDT/栈**:每个 AP 独立(不能共享 BSP 的,否则 TSS rsp0 互踩)。
6. **lost-wakeup(P1 已修?否,Phase 3)**:Phase 1+2 只启动 AP + per-CPU,**不修 futex/waitpid 并发**(那留 Phase 3 或 M4)。单核 + AP idle(不跑任务)可避免暴露。若 AP 跑任务,M4 调度并发再修。
7. **Spinlock 已原子**(test-and-set,多核正确,只是不公平)——够 M3 启动;ticket lock 留 M5。
8. **-smp 2 测试**:`timeout 40 cmake --build build --target run-kernel-test`(qemu.cmake 加 -smp?可能要改 qemu.cmake 或 test target)。或 make run -smp 2 冒烟。
