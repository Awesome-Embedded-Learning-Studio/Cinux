# F3-M1 信号系统

> 2026-06-18。从零构建 POSIX 信号（核心 22 个）：投递/处理/屏蔽 + kill/sigaction/sigprocmask/sigreturn + Custom handler round-trip + PF→SIGSEGV/exit→SIGCHLD/write→SIGPIPE 集成。5 批，763→783（+20 测试），fresh 783/0 + 实机 GUI 冒烟启动到桌面。

## 范围与切分（propose 已确认）

- 批1 数据结构：Signal enum / SigSet bitmask / SigAction + Task 信号字段 + fork 继承（清 pending）。
- 批2 投递逻辑：signal_send / signal_pick_deliverable / signal_exec_default / signal_check_and_deliver + pid→Task 注册表 + kill/sigaction/sigprocmask syscall。
- 批3 signal frame + sigreturn：中断路径投递 Custom handler + int $0x80 trampoline + sigreturn gate。
- 批4 集成：PF→SIGSEGV + exit→SIGCHLD + write→SIGPIPE。
- 批5 收尾：libc wrapper + 文档 + 实机冒烟。

## 关键架构决策

### 1. Custom 投递走中断路径，不改 syscall.S

`syscall.S` 只保存精简帧（user_rsp/rip/rflags + 6 参数 + rbx/rbp，**无 R12-R15**），sigreturn 无法经 syscall 完整恢复用户上下文。两个方案：改 syscall.S 到完整 frame（高风险，影响所有 syscall）vs 用 int $0x80 软中断走完整 InterruptFrame。**选后者**：

- Custom signal 在中断/异常返回路径投递 —— ISR 宏在 `call handler` 后调 `signal_check_deliver_isr(frame)`，Custom 时 `signal_setup_frame` 在用户栈构造 SignalFrame + 改 frame→handler。
- sigreturn 经 IDT vector 0x80 trap gate（DPL=3）进入，收完整 InterruptFrame 恢复。
- syscall 路径（`signal_check_and_deliver`）只投递 Default/Ignore，Custom 留 pending 给下次 IRQ0（时钟中断频繁，延迟可忽略）。

### 2. sigreturn trampoline 栈注入

handler 返回地址指向栈上的 `int $0x80`（cd 80 + nop，8B 槽）。依赖栈页可执行 —— **NXE 未启用（GOTCHA#10）故可行**。F9 启用 NXE 时栈不可执行，trampoline 失效，必须迁 vdso / 独立可执行页。

### 3. signal_check_deliver_isr 只对用户态投递

严判 `frame->cs & 0x03`。kernel-test 全 ring0（中断点 cs=kernel）→ skip，保护批2 测试设的 pending 栈 task 不被中断误投递（exit_current 会切走栈 task 崩）。实机 ring3 中断点才投递。

### 4. SIGCHLD 投递 + waitpid non-blocking

exit 时 `signal_send(parent, SIGCHLD)`。SIGCHLD default=Ignore（`signal_exec_default` kIgnore no-op），不杀父。waitpid 保持 non-blocking（轮询），阻塞/唤醒留 TODO —— 无 Stopped 状态机 + wait_queue，提前做空地基。

## 集成点

- **PF→SIGSEGV**：`handle_pf` user-fault no-VMA 不再 `exit_current`，改 `signal_send(SIGSEGV) + return`，ISR signal check 投递（可被 handler 捕获或默认终止）。kernel-test ring0 不走此分支（GOTCHA#11 `err&0x04`）。
- **exit→SIGCHLD**：`sys_exit` 设 Dead 前 `signal_send(parent, SIGCHLD)`。
- **write→SIGPIPE**：`sys_write` write 失败 `errno==EPIPE` 时 `signal_send(self, SIGPIPE)`。

- 单测 +20：SigSet ops / 默认动作表 / uncatchable / valid / Task 字段（批1，7）；signal_send（忽略/不可捕获）/ pick（blocked/Custom skip）/ sigaction/sigprocmask/sys_kill self（批2，10）；signal_setup_frame 构造 / sigreturn 恢复（批3，2）；sys_kill 跨进程经 registry（批4，1）。

- 实机：生产内核 `make run` 启动到 GUI 桌面（Desktop icons / gui_worker），`signal_check_deliver_isr` 在每个中断（PIT/AHCI）后高频跑全程不炸；`kernel_init exit→SIGCHLD` 投递不炸。**Custom handler 真用户态 round-trip 留后续**（libc wrapper 已就绪，需用户程序触发）。
