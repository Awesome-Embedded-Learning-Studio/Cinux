# F8-M2 批3 — 命名 FIFO 核心（FifoRegistry + FifoOps cloning open）

> 2026-06-30，worktree `worktree-f8-pipe-fifo`。F8-M2 第三批（FIFO 机制 + host 单测）。

## 背景 / 目标

给匿名 Pipe 一个文件系统名字（Linux `mkfifo` / `S_IFIFO`）。mkfifo 注册名字；open 该名字克隆出
绑定共享 Pipe 读/写端的 per-open inode。复用 F8-M1 增强后的 Pipe（真调度阻塞 + O_NONBLOCK）。

## 设计 / 决策

**FifoRegistry**：内存型固定表（`Entry{name, used, Fifo}` ×16），name→Fifo 映射；singleton
（`FifoRegistry::instance()`）。`create/lookup/remove`，spinlock 守护。无 `<map>/<string>`（freestanding）。

**Fifo**：持 `Pipe* pipe{nullptr}` + 自带 Spinlock；`get_or_create_pipe()` 首次 open 懒建 Pipe（race-safe，
两个 open 并发首建只造一个）。

**FifoOps**（InodeOps 子类，singleton `fifo_ops()`）：`open(inode, flags)` cloning——
- `inode->fs_private` 指 `Fifo*`（per-FIFO 状态收进 fs_private，**不改 InodeType enum**）。
- 首次 open 懒建 Pipe；按 flags 访问模式选 `PipeReadOps`（O_RDONLY）vs `PipeWriteOps`（O_WRONLY）；
  O_NONBLOCK 透传给 pipe 端。
- 返 per-open inode（ops 是新 `PipeReadOps/PipeWriteOps`）。

**关键接口变更**：`InodeOps::open(Inode*)` → `open(Inode*, uint64_t flags)`。FIFO 的 open 必须知道
方向（读/写端），flags 是唯一途径。查牵连后 blast radius 干净：1 override（PTY `PtmxOps::open`，忽略
flags）、1 caller（`sys_open` do_open_kernel 传 flags）、1 default（inode.cpp，忽略 flags）。**PTY 零行为变**
（open 仍无条件克隆）。

## 范围栅栏（诚实标注，不包装）

- **open()-级阻塞（两端都 open 前 open 阻塞）推迟**：今日 writer 可在 reader open 前缓冲至多 4KB，
  数据级阻塞（pipe wait queue）仍保正确性。Linux-faithful open 阻塞留真 shell 闭环。
- **close 不销毁 pipe 开新 epoch**：无 `InodeOps::release` 钩子，re-open 复用旧 pipe（同匿名 pipe 的
  「close fd 释放 File 不释放 ops/pipe」hobby-OS 限制）。
- per-open ops/inode 在 close 泄漏（同上限制）。
- FIFO 经 DevFS 命名节点落地 + sys_mknod 在批4；本批纯逻辑 + host 单测。

## 陷阱

**fifo.hpp 在 `namespace cinux::ipc`**：`Spinlock`（在 `cinux::proc`）和 `struct stat`（在 `cinux::fs`）
都要**全限定**——devfs.cpp 能用非限定是因为它本身在 `cinux::fs`。首编报 `Spinlock does not name a type` /
`stat incomplete type` / `marked override but does not override`。

## 验证

- host `test_fifo` 4 例：registry create/lookup/remove + 重复→AlreadyExists + 空→InvalidArgument；
  首 open 懒建唯一共享 pipe + read/write round-trip；O_NONBLOCK 透传（满 pipe 写→WouldBlock）；
  stat 报 kSIfFifo + 无 fs_private open→InvalidArgument。
- `run-kernel-test-all` 两 leg **990/0**（批3 不加内核测，留批5 跨 fd round-trip；证 open() 签名变更零回归：
  PTY 测全绿）。
- host 相关测全绿：pipe / sys_pipe / devfs / pty / fd_table / vfs_mount / fifo。
