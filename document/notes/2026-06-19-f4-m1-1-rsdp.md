# F4-M1 批1:ACPI RSDP 定位与 checksum 校验

> 2026-06-19。F4(SMP)里程碑 M1(ACPI 静态表解析)批 1。
> 范围:ACPI 结构体 + RSDP 发现 + checksum + 单测。不解表(M1-2)/ MADT(M1-3)/ main 接线(M1-4)。
> 基线 main = 840/0(912b623,F3-M4 + F-INFRA 收官)。

## 目标
从 BIOS 提供的 ACPI 表定位 RSDP(Root System Description Pointer)并校验 checksum,为 M1-2 的表遍历(RSDT/XSDT → MADT/FADT/HPET)提供入口。这是 F4 多核/SMP 的地基:APIC 地址、CPU 列表全来自 ACPI。

## 决策
- **物理地址经 direct-map 访问**:RSDP(<1MB)/ SDT(低 RAM)经 `phys_to_virt(phys) = DIRECT_MAP_BASE + phys`,复用 mini loader 的 identity 映射(全 RAM)。与 APIC MMIO(M2,需单独 `VMM.map + FLAG_PCD` 禁缓存)严格区分——ACPI 表是普通 RAM,cache-enabled direct-map 正确。
- **ACPI 1.0 + 2.0 双校验**:revision 0 只校验前 20 字节;revision >= 2 额外校验整个 `length`(extended_checksum)。
- **EBDA + BIOS ROM 双搜索**:先 EBDA(物理 0x40E 的段指针 << 4,前 1KB),后 0xE0000-0xFFFFF,16 字节对齐步长(ACPI 要求)。

## 实现
- `kernel/drivers/acpi/acpi.hpp`:`RSDP`(36B)+ `SDTHeader`(36B)`[[gnu::packed]]` + `static_assert` 布局锁 + `validate_checksum`/`find_rsdp` 声明 + `kRsdpSignature`。
- `kernel/drivers/acpi/rsdp.cpp`:`validate_checksum`(逐字节 mod 256)+ `find_rsdp`(EBDA/ROM probe + signature 比对 + 双 checksum)+ `phys_cast<T>` direct-map 模板。
- `kernel/test/test_acpi.cpp`:9 测(4 checksum 纯逻辑 + 5 RSDP 真机)。
- CMake 接线:`drivers/CMakeLists.txt` 加 `acpi/rsdp.cpp`;`kernel/CMakeLists.txt` test 列表加 `test_acpi.cpp`;`main_test.cpp` 注册 `run_acpi_tests()`(在 keyboard 后,中断/发现组)。

## 验证
- `timeout 40 cmake --build build --target run-kernel-test`:**849 passed, 0 failed**(840 基线 + 9 ACPI)。
- 零编译警告(F-INFRA 基线;-Wold-style-cast:诊断打印用 `static_cast` 非 C cast)。
- 真机日志:`[ACPI] RSDP found: rev 0, rsdt 0xBFFE239C`(4 次,每次 find_rsdp 调用)。

## 关键发现(GOTCHA,影响 M1-2 / M2)
1. **QEMU pc 默认 ACPI 1.0 RSDP(revision 0)**,只有 32 位 `rsdt_address`(实测 0xBFFE239C),**无 XSDT**(`xsdt_address` 是 reserved 字段,读出 0)。⇒ M1-2 表遍历**主路径必须是 RSDT(32 位指针数组)**,XSDT(64 位)仅作 revision>=2 的可选分支。初版测试断言 `revision>=2`/`xsdt!=0` 据此失败(实测暴露),已修正为 `revision==0 || >=2` + 改验 `rsdt_address`(所有版本都有)。这条修正了 propose 阶段"QEMU 提供 ACPI 2.0"的错误假设。
2. **测试 log 含 ANSI escape → grep 当二进制静默**:QEMU 串口 + make 输出混 escape sequences,`file` 报 "ASCII text, with escape sequences",`grep` 默认判定二进制后**既不输出匹配也不报 "Binary file matches"**(完全静默),导致首次以为"ACPI 段没运行"。诊断/查日志一律用 **`grep -a`** 强制文本模式。F-INFRA 后查 QEMU 串口日志的同款陷阱。
