# F9 批9 — M3 进程凭证(uid/gid/euid/egid + getuid/setuid syscall)

> 2026-06-26 / F9 安全机制 / 批9(M3 凭证)。分支 `feat/f9-security`,commit `75ad715`。
> **F9 全域收官(M1-M4 全 ✅)**。范围(用户拍板「标准档」):4 字段 + fork 继承 + 6 syscall,setuid 简化语义。

## 背景:让批9 比想象轻的免费午餐

调研发现 fork.cpp:128 / clone.cpp:148 创建子进程是 `memcpy(child, parent, sizeof(Task))` 整体拷,拷完只覆盖 tid/pid/tgid/ppid/pgid/sid 等非凭证字段。**所以 Task 一旦加 uid/gid 字段,fork/clone 自动继承,零改 fork/clone 代码**(标量字段 memcpy 直接拷对)。TaskBuilder `new Task` 默认初始化 uid=0(root),kernel init task → fork shell 链路自动全是 root。

inode 已有 uid/gid/mode 字段(inode.hpp:86-101,ext2 存的),stat 返 st_uid/st_gid。文件侧凭证地基在,M3 不动文件。无 check_permission(留 F6)。故本批做完所有进程都是 root——没真权限隔离,但 M3 目标是凭证基础设施,隔离是 F6 的事。

## 改动

- **process.hpp**:Task 加 `uid`/`euid`/`gid`/`egid`(uint32_t,默认 0=root),controlling_tty 后。注释说明 fork memcpy 继承 + setuid binary/saved-set 留 F6。
- **fork/clone**:零改(memcpy 覆盖,覆盖段不碰凭证)。
- **execve**:零改(setuid binary 留 F6)。
- **errno.hpp**:加 `kEperm`(1,Linux EPERM;setuid 失败语义,区别于 kEacces=13 文件权限)。
- **sys_creds.{hpp,cpp}**(新,仿 sys_pgrp 多 handler):getuid/geteuid/getgid/getegid 直返字段;setuid/setgid 简化语义——root(euid/egid==0)可设任意,非 root 只能回退 real(`euid==0 || new==uid`)。返 0/-EPERM。
- **syscall_nums.hpp**:加 6 Linux x86_64 标准号(getuid=102/getgid=104/setuid=105/setgid=106/geteuid=107/getegid=108)+ syscall.cpp register_builtin 注册。
- **user/libc/syscall.{cpp,h}**:6 syscall 包装(`_syscall1`)。
- **test_creds.cpp**(新):默认 root / get 返当前 / setuid root 全能 / 非 root EPERM + 回退 real / setgid 镜像 / **fork memcpy 继承实证**(`__builtin_memcpy` 整个 Task,验 uid/euid/gid/egid 拷贝——production fork 继承路径单测化)。

## 牵连核实(改前 grep)

- Task 加字段在 controlling_tty 后,ctx/on_cpu/pgid 等关键字段偏移不变 → context_switch.S 依赖 `ctx@0` + `on_cpu` 偏移的 static_assert(process.hpp 296)仍过,asm 不受影响(编译期坐实)。
- syscall 体系对齐 Linux 号(getpid=39 等),6 个凭证号 < SYSCALL_TABLE_SIZE(256)。

## 诚实记录(GOTCHA)

- **make run VNC 5900 外部占用**:端口 5900(VNC :0)被非-QEMU 进程 LISTEN,make run 的 QEMU 启动失败(`-vnc :0: Failed to find an available port: Address already in use`)。临时 `sed qemu.cmake -vnc :0→:1` 跑 production 冒烟(零 panic),跑完 `git checkout cmake/qemu.cmake` 还原。非代码问题,环境冲突。
- **test log grep -a**(F4-M1 GOTCHA 复现):log 含 ANSI escape,grep 当二进制静默。
- 所有进程目前都是 root(uid 0):没 multi-user,没 setuid binary,没 check_permission。真权限隔离在 F6。M3 交付的是「凭证字段 + 查改 syscall + 继承」基础设施。
