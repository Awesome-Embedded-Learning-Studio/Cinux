# F4-M2 批3:PIC→APIC 切换(中断现代化)

> 2026-06-19。F4-M2 批 3(M2 核心 + F4 最高风险)。范围:EOI 抽象 + PIC→APIC 切换 + 改 4 handler EOI + main 接线 + 真机冒烟。
> 基线 = M2-2(869/0)。**F4 中断现代化落地:8259 PIC → LAPIC + IOAPIC**。

## 目标
把系统中断从 8259 PIC 切到 Local + I/O APIC,解锁现代中断(更多中断源 + 为 M3 多核 / xHCI INTx# 铺路)。改全局中断路径 + 真硬件,F4 最高风险批。

## 决策
- **EOI 抽象层(IrqBackend)**:`irq_eoi(irq)` 按 `g_irq_backend` 派发(PIC→`send_eoi` / APIC→`LAPIC.eoi`)。handler 一处切换,**不动 interrupts.S**(EOI 在 C handler 发,stub 不含 EOI)。
- **IOAPIC 重定向到 PIC 同向量(0x20/0x21/0x2C)**:IRQ0/1/12 经 ISA override 查 GSI(QEMU IRQ0→GSI2),IOAPIC `set_redirect(gsi, vector, bsp=0)`,**复用现有 16 stub + handler**,只 EOI 源切 LAPIC。
- **切换在 sti 前(IF=0 原子)**:mask PIC → LAPIC init + enable(spurious 0xFF)→ IOAPIC init + redirect + unmask → flip `g_irq_backend` → sti。
- **失败降级**:无 ACPI / IOAPIC map 失败 → 留 PIC(不崩)。
- **APIC MMIO 经 `VMM.map + FLAG_PCD`**(M2-1/2),非 direct-map。

## 实现
- `irq_backend.hpp/cpp`(新,arch):IrqBackend enum + `g_irq_backend` + `irq_eoi`(派发)+ `switch_to_apic`(mask PIC / LAPIC init+enable / IOAPIC redirect IRQ0,1,12 / flip)+ `isa_irq_to_gsi`(查 override)。
- 改 5 处 EOI(pit:0 / keyboard:1×2 / mouse:12 / irq_handlers default:0 + mouse_off:12):`PIC::send_eoi` → `cinux::arch::irq_eoi` + include。
- `main.cpp`:Step 17 `PIC::unmask(0/1/12)` → `switch_to_apic()`(sti 前)。
- `arch/CMakeLists.txt` 加 `irq_backend.cpp`。

## 验证
- `run-kernel-test`:**869/0**(test PIC mode,handler irq_eoi PIC 转发)。零编译警告。
- production `make run` 冒烟:**`[APIC] switched to APIC (LAPIC id 0, IOAPIC @0xFEC00000)`** + GUI 启动不崩。
- **PIT tick 探针(临时):13 ticks under APIC**(100 Hz,~130 ms spin)—— 证明 IRQ0 → IOAPIC GSI2(override)→ LAPIC vector 0x20 路由通,PIT timer 在 APIC mode 正常。探针验证后删除。

## 关键
- 切换成功 + PIT 中断工作 = M2-3 核心达成。8259 PIC 已退役(masked),中断走 LAPIC/IOAPIC。
- GOTCHA:freestanding 下 `for (x : {...})` 需 `<initializer_list>`(改普通数组循环避);IOAPIC redirect 必须照 ISA override 查 GSI(QEMU IRQ0→GSI2,直映 IRQ0→GSI0 会错,无 tick)。
