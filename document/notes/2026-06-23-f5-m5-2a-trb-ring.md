# 2026-06-23 F5-M5 批2A — TRB + ring 数学（纯）

## 背景 / 目标

xHCI 核心数据结构:TRB（16B）+ 两种 ring。生产者 ring（命令/传输,host 生产）:末尾 Link TRB 闭环 + 每圈翻 PCS。消费者 ring（事件,控制器生产）:**无 Link TRB**,按 ERST size 回绕 + 翻 CCS。纯（操作 Trb* 数组,无 DMA/VMM）,host 直测;kernel 在 2B 用 DmaBuffer 背书 + 写 CRCR/ERSTBA。

## 变更

- xhci_trb.hpp（NEW）:`Trb`（16B static_assert）+ `TrbType`（命令/传输/事件）+ 控制位常量（kCycleBit/kToggleCycle/IOC/IDT/CH/kTrbTypeShift）+ `trb_control`/`trb_type` helper。
- xhci_ring.hpp/.cpp（NEW）:`TrbRing`（init/enqueue/PCS flip/Link TRB）+ `EventRing`（init/dequeue/CCS flip）。纯。
- test_xhci.cpp（host）:+5 ring 用例（enqueue 设 cycle=PCS、Link TRB 回绕翻 PCS、第二圈用翻转 cycle、EventRing 按 CCS 出队、CCS 回绕翻转）。
- 构建:drivers CMake 加 xhci_ring.cpp;test_xhci 改 integration test 链 xhci_ring.cpp。

## 关键陷阱

- **命令/事件 ring 回绕机制不同**:命令/传输 ring 用 Link TRB（末尾,type Link + ToggleCycle,host 翻 PCS）;事件 ring **无 Link TRB**,按 ERST 段大小回绕 + 翻 CCS。两类分开实现（TrbRing vs EventRing）,别混。
- **Trb 成员 volatile**:`EventRing::dequeue` 成员逐个拷贝（避免 volatile-struct 整体赋值歧义）。
- **using-decl 不能命名 namespace**（TrbType）:test 改 `using namespace usb`。

## 验证

- host:test_xhci **11 例**（6 寄存器 + 5 ring）全过。
- run-kernel-test:**929/0**（xhci_ring.cpp 进 kernel,无调用方,dead code 无害）。run-kernel-test-xhci 同 929/0（无新 kernel 行为）。
- ring 与真硬件（DmaBuffer+CRCR/ERSTBA）集成留 2B/2C。

## 遗留

- 2B:DCBAA + scratchpad + 中断器/ERST + USBCMD.RS 启动（DmaBuffer 背书 + phys 写寄存器）。
