# Cinux v1.0.0-rc1 — 🎉 First Candidate Release

**发布日期:2026-07-13**

从 Bootloader 到 GUI 桌面、从内核到 GCC 自举,全链路打通的首个稳定版。**13 个 Feature 域主体完成**,目标平台 QEMU / WSL2 KVM(BIOS 启动;UEFI 推 v1.1+)。

> 格式参照 [Keep a Changelog](https://keepachangelog.com/zh-CN/),版本号遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## ✨ Highlights

- **🧠 完整 x86_64 内核**:Bootloader(实模式 → 保护模式 → 长模式两阶段加载)→ Mini Kernel → Big Kernel → Ring 3 用户态 → GUI 桌面,全链路实现。
- **⚡ SMP 多核**:`-smp 2` 双核 online。per-CPU 架构(GS/swapgs/per-CPU GDT·TSS)+ IPI + trampoline + AP boot + per-CPU runq + 多核调度 + reschedule IPI + lockdep 锁序图死锁检测(opt-in)。
- **📁 多文件系统 + 三套存储**:ext2 读写(独立库 `libs/ext2/`)+ ext4 只读(extent)+ tmpfs + ProcFS + DevFS;块设备 **NVMe / VirtIO-blk / AHCI** 三套驱动;Dentry Cache 路径加速 + POSIX `flock` + 运行时 `sys_mount`。
- **🌐 TCP/IP 协议栈**:以太网 / ARP / IPv4 / ICMP / UDP / TCP(三次握手 / 序号-ACK / 四次挥手 / RST)+ Socket API(per-socket 环 + 阻塞 + accept 队列)。真 `ping 10.0.2.2` 干通。
- **🖥️ GUI 用户态桌面**:[Cinux-GUI](https://github.com/Awesome-Embedded-Learning-Studio/Cinux-GUI)(独立 submodule 仓)Compositor + Widget 树 + TerminalWidget,以**用户态进程**运行(`/dev/fb0` mmap + `/dev/input` 事件),xHCI USB HID 键鼠,多终端窗口。
- **👨‍💻 GCC 自举**:Cinux 上 `gcc hello.c && ./a.out` / `g++ hello.cpp`,cc1 → as → ld → ./hello 全闭环;默认 PIE;syscall unhandled 90 → 0;busybox init 作 PID 1。
- **🔌 musl 动态链接**:移植 musl 作唯一 libc(不自建 libc 生态),kernel 做 PT_INTERP / interp 加载 / auxv,跑通 musl 动态 hello。
- **🔒 安全机制**:NXE + SMEP + SMAP(均真生效并通过机制回读验证)+ ASLR + UID/GID + Stack Canary(`-fstack-protector-strong` + TSC canary)。
- **🔧 现代 C++17**:`constexpr` 编译期生成 GDT/IDT,RAII 锁管理,`enum class` 驱动接口,freestanding 零标准库依赖,`ErrorOr<T>` 无异常错误传播。
- **🧪 测试驱动**:`run-kernel-test-all` 单核 + `-smp 2` 两 leg ~1946 项全绿;host 单测 + TSAN/ASAN/UBSAN/LOCKDEP sanitizer 矩阵;机制回读验证硬件真生效。

## ⚠️ Known Issues

- **NVMe 偶发 I/O 错误(status=0x4080,LBA 越界)**:根因(为何 NVMe 命令 LBA 越界)持续取证中(CID 关联 + QEMU invalid-lba trace 已就位)。**已兜底**:文件页读失败对用户态投 SIGBUS、执行到 garbage 投 SIGILL —— **单个用户进程死,内核不再 panic**。
- **ext4 只读**:读路径完整,写 / journal 推 v1.1+(ext2 已支持读写)。
- **epoll 未实现**:`poll` / `select` 可用。
- **TCP 最小可用**:无重传 / RTO / 窗口 / 拥塞控制(loopback + 局域网够用)。
- **DEBT-021**:xHCI `poll_events` 并发(P1,留 xHCI 重构)。
- **UEFI 启动**:推 v1.1+,BIOS bootloader(MBR + stage2)够用。
